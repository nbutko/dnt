Roadmap & Milestones

Something playable at every step, in build order.

Each milestone below produces something that can actually be handed to him to try — no milestone is "invisible plumbing only." Scope is deliberately small per step; expand once each layer is proven fun.

M0 — Combat math spike playable: yes (barebones)

Build the battle math (damage formula, HP, timers, monster typing simulation) as plain functions with no UI polish — a single unstyled page: one text box, one monster with a health bar, one player health bar. Prove the numbers feel right before investing in any art or animation.

  • Type a line, see damage happen, see the monster's bar tick down on its own clock.
  • Tune baseDamage, the baseline-WPM time limit, and monster wpm/accuracy/attention until a fight feels winnable-but-tense at ~30-45 seconds.

Detailed build breakdown — file/component layout, state architecture, and stories/tasks — in the M0 implementation plan.

M1 — Playable vertical slice playable: full battle

One real UI: prompt display, countdown, HP bars for both sides, win/lose screen. Hardcoded to one monster (Slime) and one text tier. No save data yet — every session starts fresh.

  • Deliverable: he can sit down, fight the Slime, and see a win or lose screen.
  • This is the first "does this even feel fun" checkpoint — get his reaction before building progression on top.

M2 — Progression loop playable: multi-battle session

Add the save file (IndexedDB), a world map (Inn, a locked Shop, and 11 habitat-themed dungeons), the hearts/lives system, coins, XP, and the skill tree. Each dungeon is a randomly-generated set of branching paths of monsters leading to a boss, gated behind clearing the previous dungeon's boss. Full breakdown — habitat difficulty order, path generation rules, hearts, the Inn — in the M2 scope doc, with the step-by-step build in the M2 implementation plan.

  • Deliverable: pick a dungeon from the map, fight your way along a branching path to its boss, earn XP, spend it at the Inn, feel the next dungeon get slightly easier to approach.
  • Skill-tree scope for M2: only the two branches whose effects are fully specced and wire into systems that exist yet — Endurance (max HP + hearts) and Wordsmith (text-tier unlocks) — are buyable. Focus, Luck, and Utility render on the tree but ship visible-but-locked ("Coming soon"), since their magnitudes aren't spec'd yet and Utility's power-up slots have nothing to fill until M3. They get their numbers and are switched on in M5 (Utility no earlier than M3, once power-ups exist).

M3 — The character sheet (the D&D layer) playable: full loop

Grew well past the original "power-ups & juice" plan: M3 brings the missing D&D concepts into the typing game — six ability scores, real 5e-style leveling (which replaces the M2 skill tree), classes, character creation, dice rolls (an encounter d20 + per-hit weapon damage dice), equippable weapons, consumable items, and the Shop (coins finally get a sink). Power-ups survive as the consumable items; the old hit-flash / screen-shake / SFX "juice" list rides along where the dice and damage numbers create natural animation moments. Full breakdown in the M3 scope doc. Expand text banks to tiers 3–4 as needed.

  • Deliverable: the full loop end to end feels like a role-playing game, not a math worksheet — you roll a character, level it up the D&D way, buy gear, and every fight opens with a d20 — all still without a single image asset.

M4 — Ship it playable: on his Chromebook, for real

Wire up hosting (GitHub Pages, via a GitHub Actions build), add the PWA manifest/service worker, install it on the actual Chromebook, and test with the actual keyboard in the actual room he'll play in.

  • Deliverable: an icon on his Chromebook that opens the game, no laptop/dev-server dependency on your machine.

M5 — Content & balance pass playable: ongoing

Playtest with him directly, watch where he struggles or gets bored, retune damage/time constants and the skill tree pacing. Add tier 5–6 content and the remaining roster (Wraith, Dragon), maybe a second boss.

  • Tune the character layer. M2's visible-but-locked Focus/Luck/Utility skill branches were retired in M3, absorbed into the six ability scores and D&D leveling (see the M3 scope). This is where that layer gets its real numbers: the XP-to-level pacing, per-modifier magnitudes, HP scale, weapon/item pricing, and the fumble/inspiration knobs — all the open tuning questions M3 shipped with placeholders for.
  • This milestone doesn't really "end" — it's the ongoing content/balance loop once the game exists.

M6 — Visual upgrade playable: stretch, optional

Once the text-only game is proven and stable, swap in real monster art/sprites and start pulling in tier 7–10 long-form content (see Content Plan: public-domain children's books, excerpted). This is deliberately last and deliberately optional — the game is already complete and playable without it.

  • Deliverable: the same battles, now with actual monster illustrations instead of text descriptions.

Suggested build order recap

  1. M0: math spike (no UI)
  2. M1: one real battle, one monster
  3. M2: save file, XP, skill tree, more monsters
  4. M3: power-ups, text/CSS-based animation, sound, more text tiers
  5. M4: hosting + PWA + Chromebook install
  6. M5: playtest-driven balance and content, forever
  7. M6: real art + long-form (tier 7–10) content, whenever it's worth the time
