⌨️ Dungeons & Typing

A monster-battle typing trainer for a 10-year-old on a Chromebook.

The pitch

The player fights a monster by typing. The player and the monster each continuously race their own line of text against their own clock, independently — type yours correctly in time and you land a hit; if the monster finishes its own line in time, it hits back. Bigger/harder text chunks hit harder. Win the battle, earn coins, XP, and loot. XP levels up a real D&D-style character — six ability scores, a class, dice rolls — raising max HP, allowed text length, and time limits as it climbs; coins buy weapons and consumable items at the Shop. So the game gets harder and the player gets visibly better at typing over time.

This is a solo side project for one specific kid, not a product for the world. That should keep scope small and choices simple — favor "works great for him, today" over "scales to a thousand users."

Guiding principles

  • Playable early. The first milestone is a single ugly battle in the browser, not a fully designed game. See Roadmap.
  • Typing quality first, game juice second. If a feature doesn't make him type more, or type harder things, it's decoration — nice, but never blocks a milestone.
  • No accounts, no backend, no login. One kid, one Chromebook (for now). Progress saves locally. See Persistence.
  • Numbers should be easy to retune. Combat math, difficulty tiers, and every ability/leveling magnitude all live in a few config files/objects (config/abilities.ts, config/leveling.ts, …), not scattered through code, so balance can change without a rewrite.
